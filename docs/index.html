Objetivo

Este documento tem o objetivo de orientar Estabelecimentos de Saúde ou Empresas Privadas que fornecem soluções/software na área de saúde a utilizarem os serviços de Armazenamento FHIR (CLOUD HEALTHCARE SMS) que foi desenvolvido pela SMS – Secretaria Municipal de Saúde do Rio de Janeiro, fornecendo as orientações técnicas necessárias para a integração dos sistemas/soluções locais com a rede.

Meio de Integração

A integração com a CLOUD HEALTHCARE SMS dar-se-á por meio dos serviços mencionados abaixo. Seguindo as boas práticas serão disponibilizados dois ambientes para a integração: homologação e produção.

Ambiente de Homologação

O ambiente de homologação tem como finalidade validar a integração, seus parâmetros de entradas, saídas e comportamentos negociais, permitindo a realização de testes antes da efetiva comunicação com o ambiente de produção. O endereço dos componentes de integração, no ambiente de homologação, é:

https://healthcare.googleapis.com/v1/projects/rj-sms-dev/locations/us-central1/datasets/raw_layer/fhirStores/FHIR_STORE_ID

Ambiente de Produção

O ambiente de produção é o ambiente estável e real que provê os serviços a serem consumidos. O endereço dos componentes de integração, no ambiente de produção, é:

https://healthcare.googleapis.com/v1/projects/PROJECT_ID/locations/LOCATION/datasets/DATASET_ID/fhirStores/FHIR_STORE_I

Verifique com a equipe da SMS as credenciais de acesso e os IDs necessários para integrar com o ambiente de produção.

Acesso ao Serviço

Para que seja possível acessar os serviços disponibilizados no CLOUD HEALTHCARE SMS é necessário entrar em contato com a SMS para que possa ser enviado as credenciais de acesso.

Resources

O CLOUD HEALTHCARE SMS segue os padrões definidos pelo projeto Rede Nacional de Dados em Saúde do SUS. Para informações detalhadas sobre a estrutura de dados, utilizar a documentação oficial do projeto:

https://rnds-fhir.saude.gov.br

A relação completa de todos os resources suportados pela plataforma até o momento é listado a seguir contendo as regras de integração.

Patient

Dados demográficos sobre um indivíduo enquanto sujeito de ações e serviços de saúde.
Padrão esperado para integração:
https://rnds-fhir.saude.gov.br/StructureDefinition-BRIndividuo-1.0.html

Operações

Google Healthcare API

O Cloud Healthcare da SMS-RIO é construindo em cima do serviço Healthcare API do Google. Instruções detalhadas sobre como integrar podem ser encontrados na documentação oficial do serviço:
https://cloud.google.com/healthcare-api/docs/reference/rest

Biblioteca para facilitar a integração

Disponibilizamos uma classe em Python para facilitar a integração com a API. Nela encontrarão a implementação das funções de CRUD (Create, Read, Update e Delete) para qualquer resouce do FHIR. Também encontrarão uma classe que permite ler e atualizar o Paciente pelo CPF.

https://github.com/prefeitura-rio/fhir-utils/blob/main/fhir_utils/healthcare_api.py

Parâmetros necessários: 
                                        PROJECT_ID = 'rj-sms-dev'
                                        LOCATION = 'us-central1'
                                        DATASET_ID = 'raw_layer' 
                                        FHIR_STORE_ID = nome da empresa
Autenticação
Para autenticação é necessária uma variável de ambiente chamada GOOGLE_APPLICATION_CREDENTIALS contendo o path para as credenciais em json recebidas através da Secretaria de Saúde.
